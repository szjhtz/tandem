# Article 50 Transparency

Article 50 of the EU AI Act includes transparency obligations for certain AI systems, including disclosure when natural persons interact directly with an AI system and marking some AI-generated or manipulated content. This document describes Tandem's intended transparency posture for generated text, plans, and artifacts.

This is implementation guidance for Tandem and deployment guidance for users. It is not a legal determination for any specific deployment.

## Current State

Tandem records runtime provenance for generated workflow plans, automation outputs, artifacts, receipts, and validation state. That provenance is useful for audit and debugging, but visible UI labeling is not yet systematic across all desktop and web surfaces.

## Intended User-Facing Label

Use the visible label:

```text
AI-Generated
```

Use `AI-Assisted` only when a human-authored artifact has been materially edited, summarized, transformed, or structured by Tandem but should not be presented as fully generated.

## Surfaces To Label

Apply visible labeling to:

- Generated chat text and assistant-authored summaries.
- Proposed workflow plans and plan previews.
- Generated artifacts, briefs, reports, and handoffs.
- Generated review recommendations and risk summaries.
- Exported artifacts where practical, either in document metadata or a small header.

## Minimum Badge Behavior

The future desktop and web implementation should:

- Show a visible `AI-Generated` badge close to the generated content.
- Provide accessible text for screen readers.
- Include tooltip or help text such as: "This content was produced or materially transformed by an AI system. Review before relying on it."
- Preserve run or artifact provenance where practical.
- Avoid implying human approval unless a recorded human review or approval exists.

## Draft, Reviewed, And Approved States

Generated content can move through several operational states:

| State                             | Meaning                                                                         | Suggested UI text        |
| --------------------------------- | ------------------------------------------------------------------------------- | ------------------------ |
| Draft                             | Generated by Tandem and not yet reviewed.                                       | `AI-Generated draft`     |
| Reviewed                          | Human reviewed but did not necessarily approve for external use.                | `AI-Generated, reviewed` |
| Approved                          | Human approval decision is recorded for the specific workflow gate or artifact. | `AI-Generated, approved` |
| Human-authored with AI assistance | Human-authored content was materially transformed by Tandem.                    | `AI-Assisted`            |

## Deployer Guidance

Deployers should:

- Decide which external users, customers, employees, or reviewers will see AI-generated content.
- Keep generated content labeled until it has been incorporated into a human-owned final record under the deployer's policy.
- Avoid removing provenance from regulated artifacts unless the final recordkeeping system preserves it elsewhere.
- Train reviewers not to treat generated drafts as final decisions.
- Record when a generated draft is approved for external communication or system-of-record update.

## Planned Tandem Work

- Add a reusable `AI-Generated` badge component for desktop and web panel surfaces.
- Apply it to generated text, proposed plans, plan previews, artifacts, summaries, briefs, and handoffs.
- Add visual and accessibility tests for the main generated-content surfaces.
- Preserve generation provenance in exported artifacts where practical.
